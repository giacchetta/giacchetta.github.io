# Luciano Giacchetta вЂ” Professional Portfolio

A multilingual professional portfolio built with **Astro 6**, featuring a modern Bento UI dashboard homepage. Showcases career history, technical credentials, certifications, and detailed case studies. Available in English, Argentine Spanish, and Uruguayan Spanish.

**Live site**: https://lucianogiacchetta.com

---

## Tech Stack

- **Framework**: [Astro](https://astro.build) v6.x
- **Content**: [MDX](https://mdxjs.com) with Astro Content Collections (Zod schemas)
- **Styling**: [Bootstrap](https://getbootstrap.com) v5.3.x
- **CSS Optimization**: [PurgeCSS](https://purgecss.com) вЂ” removes unused Bootstrap at build time
- **i18n**: Astro built-in i18n routing + AI-powered translation script
- **Sitemap**: `@astrojs/sitemap` with locale support
- **Agent/LLM access**: `astro-llms-md` вЂ” generates per-page `.md` files, `/llms.txt`, and `/llms-full.txt` at build time
- **Deployment**: GitHub Actions в†’ GitHub Pages

## Localization

The site supports three locales:

| Locale | URL prefix | Notes |
|---|---|---|
| English (`en`) | none | Default, e.g. `/`, `/[slug]` |
| Argentine Spanish (`es-ar`) | `/es-ar/` | e.g. `/es-ar/[slug]` |
| Uruguayan Spanish (`es-uy`) | `/es-uy/` | e.g. `/es-uy/[slug]` |

Translated MDX files use a naming convention: `example.es-ar.mdx` and `example.es-uy.mdx` alongside the English `example.mdx`. The translation script generates these automatically from the English source using an OpenAI-compatible API.

UI strings live in `src/i18n/en.json` (English source). Run `npm run translate` to regenerate all translated files.

## Content Collections

### `collaborations`
Work history entries with two subtypes:
- `type: "company"` вЂ” An employer entry (role, period, location, summary, logo)
- `type: "article"` вЂ” A case study linked to a company via the `company` field

### `credentials`
Technical skill deep-dives rendered as full MDX pages.

### `certifications`
Professional certifications with provider, level, status, and credential URL.

## Deployment

Pushes to `main` trigger the GitHub Actions workflow (`.github/workflows/static.yaml`):
1. Install dependencies (`npm ci`)
2. Build with `npm run build:no-translate` (PurgeCSS в†’ Astro build)
3. Deploy `dist/` to GitHub Pages

Translation runs locally and generated files are committed to the repository.
